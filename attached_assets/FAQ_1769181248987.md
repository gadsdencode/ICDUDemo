# Frequently Asked Questions (FAQ)

## General

### What is ICDU?
ICDU (Intent-Conscious Data Unit) is a structured training/evaluation record format that encodes user intent, governing principles, persona, and context. It is part of a comprehensive AI safety and evaluation pipeline. See `docs/OVERVIEW.md` for more details.

### What is the purpose of this repository?
This repository is published for **evaluation and research purposes only**. It allows researchers, evaluators, and organizations to understand, test, and benchmark ICDU-based systems without requiring a commercial license.

### What components are included in the ICDU system?
The ICDU system consists of four main components:
1. **ICDU** - The structured data unit format
2. **AI Judge** - Quantitative evaluator scoring outputs
3. **HITL Nuance Grader** - Qualitative human-in-the-loop evaluator
4. **Scenario-Perturbation Stress Engine** - Generates controlled variations for testing

See `docs/OVERVIEW.md` for detailed information.

## Licensing & Commercial Use

### Can I use ICDU commercially?
Commercial use **requires a license**. See `compliance/COMPLIANCE.md` for detailed definitions of what constitutes commercial use.

### What counts as commercial use?
According to `compliance/COMPLIANCE.md`, commercial use includes:
- Deployment in a production system
- Use in a paid or monetized product or service
- Internal use that supports revenue-generating operations
- Model training or fine-tuning for commercial delivery
- Offering ICDU-based evaluation as a service

### What is permitted without a license?
As stated in `compliance/COMPLIANCE.md`, the following non-commercial evaluation uses are permitted:
- Academic research
- Internal testing without revenue impact
- Benchmarking and comparison
- Due diligence and vendor evaluation

## Enforcement

### How is ICDU usage detected?
According to `compliance/COMPLIANCE.md`, ICDU usage is detectable via:
- Required schema fields
- Evaluation report signatures
- Pipeline invariants described in `spec/`

### What happens if I use ICDU commercially without a license?
As described in `ENFORCEMENT.md`, most enforcement begins as a **clarification and licensing discussion**. Formal action is reserved for unresolved commercial use. The intent is to enable open evaluation, encourage responsible adoption, and ensure commercial use is properly licensed.

### What is the enforcement posture?
Per `ENFORCEMENT.md`, the project aims to:
1. Enable open evaluation
2. Encourage responsible adoption
3. Ensure commercial use is properly licensed

Enforcement typically starts with outreach and clarification, not immediate legal action.

## Technical

### What is the ICDU schema?
The ICDU schema is defined in `spec/icdu.schema.json`. This JSON schema specifies the required structure and fields for ICDU records.

### How do I validate an ICDU record?
You can validate ICDU records using the provided validation tool:
```bash
python tools/icdu_validate.py examples/icdu/icdu_example.json
```

### What are the required fields in an ICDU record?
Refer to `spec/icdu.schema.json` for the complete schema specification. The schema includes required fields for governance identifiers and other structural elements.

### Are there example ICDU payloads?
Yes. Example ICDU payloads are available in `examples/icdu/`. These can be used as reference implementations.

### What evaluation report formats are supported?
Evaluation report formats are documented in `benchmarks/EVALUATION_REPORT_FORMAT.md`. ICDU-conformant systems are expected to produce evaluation artifacts consistent with published formats, as noted in `ENFORCEMENT.md`.
