# Overview: ICDU-Based AI Safety + Evaluation Pipeline

This document explains how the four components fit together into a safety + promotion pipeline.

## Components
1) **ICDU (Intent-Conscious Data Unit)**
- Structured training/eval record encoding:
  - user intent,
  - governing principles,
  - persona,
  - context.

2) **AI Judge (Quantitative Evaluator)**
- Scores outputs using gates such as:
  - Intent-Alignment Score (IAS),
  - Principle-Adherence Score (PAS),
  - Application Score (AS).

3) **HITL Nuance Grader (Qualitative Evaluator)**
- Human rubric scoring for:
  - empathy, clarity, coaching quality, perceived trust.

4) **Scenario-Perturbation Stress Engine**
- Generates controlled variations (role/tone/constraints/channel)
- Measures stability, fairness, and failure modes.
